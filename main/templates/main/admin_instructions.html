Admin Instructions


Cycles

  • Cycles represent the container for a unit of evaluation data recorded during a specified period of time.
    • For example, a cycle that was started in Spring 2016 and ended in Fall 2018 contains all of the data (rubric templates, rubric scores, outcomes, measures, evaluators, etc) that was recorded/created during the time the cycle was running.
  • Cycle reports can be generated to enable the interpretation of data trends, outliers that may be the cause of lower average scores, and other statistics that could be the cause of a programs performance

  1. Select 'Create New Cycle' on the user dashboard
  2. In the year field input the year the cycle will start
  3. In the semester field input the semester the cycle will start
  4. Select 'Start'

  • A Running cycle is a cycle that data is currently being recorded within.
  • There can be more than one running cycles at a time.

  • If there is one running cycle, all actions performed on the dashboard, rubric, and outcome tabs will be done within that cycle. It does not have to be selected.
  • If there are multiple running cycles, follow the instrustions:
    1. Select 'Visit' by the desired running cycle
    2. A page for the cycle will be presented. The outcomes and their data associated with that cycle will be viewable.
  • The cycle that was worked in most recently during the last session will be the cycle that the data displayed on the dashboard and the actions performed in the rubric and outcome tabs in the next session will be associated with. Unless another running cycle is selected during the next session.

  • If the does not have any outcomes
    1. Find 'Running Cycles' on admin dashboard
    2. Select 'Visit' by the desired cycle
    3. Add the outcomes
      1. To create new outcomes individually: Select the '+' to add/create outcomes
      2. To add all of the outcomes from a previous cycle: Select 'Migrate'
        • The migrated outcomes contain the rubrics and measures associated with them in the previous cycle
        • It will not contain the students, evaluators, scores from the previous cycle
        • Migrating outcomes from a previous cycle prevents the admin from having to create new outcomes everytime a cycle is started
        • Once migrated, undesired outcomes can be deleted and new ones can be added; this does not affect the cycle the outcomes were migrated from

  1. Find 'Previous Cycles' on the user dashboard
  2. Select 'Visit' by the desired cycle
  3. The start & end dates, creator, and outcomes associated with the cycle are displayed
    • Previous outcomes can be viewed by selecting them

  1. Find 'Previous Cycles' on the user dashboard
  2. Select 'Visit' by the desired cycle
  3. Select 'Reactivate' to reactivate the cycle

  1. Find 'Running Cycles' on the user dashboard
  2. Select 'Visit' by the desired cycle
  3. Select 'End Cycle' to end the cycle



Outcomes

  • An outcome is the description for a specific criterion that a program must meet.
    • Example: Students can communicate effectivley
  • An outcome is associated with various measures, which use tools (rubrics, test scores) to provide the physical evidence of a program's success

The creation of an outcome can occur in various ways:
  • It is possible to already be working in the desired cycle
    1. Select 'Outcome' on the navigation bar
    2. Select '+' by the 'Outcomes' heading
  • A cycle could have just been created and outcomes need to be created for it
    1. Select '+' by the 'Outcomes' heading
  • The cycle for the new outcome must be accessed before the outcome can be created
    1. Find 'Running Cycles' on the admin dashboard
    2. Select 'Visit' by the desired cycle
    3. Select '+' by the 'Outcomes' heading
After following the steps for one of the scenarios above:
  1. The '+' should have been selected triggering a popup menu
  2. Fill out the Title field with the outcome title
  3. Fill out the Description field with the outcome description
  4. Select 'Add'

  1. Find the cycle the outcome is associated with (it can be a previous cycle or running cycle)
  2. Select 'Visit' by the desired cycle
  3. Select the desired outcome to access its measures, evaluators, tools, etc

  1. Select 'Outcomes' in the navigation bar
  2. Select the desired outcome
  3. The status of each measure associated with the outcome is displayed



Measures

  • A mechanism used to link a tool to an outcome in order to achieve proving the success/failure of an outcome
  • A statement of conditions that a specific type of tool will meet
  • Example: At least 75% of students will receive an average rubric score of 3 of greater when evaluated using the oral communication rubric.
    • The measure sets a standard that a specific tool will meet
    • The measure passes if the standard is met, which will contribute to the success of the outcome

  1. Access the outcome the measure will be associated with
    • Check out 'Access Outcome' under the Outcome heading
  2. Select '+' to add a measure
  3. Fill out the Measure Title field with the measure title
  4. Fill out the Cutoff-percent field with the cutoff-percent
    • The cutoff-percent in the percent of students that must meet the cutoff score
    • Example: At least 75% of students will receive an average rubric score
  5. Select the cuttoff-score type
    • Will the students be expected to achieve and average score, score in a minimum percentile, or score a minimum percent
    • Percentage Example: Students will answer correctly 60% or more of questions
    • Average Example: Students will have an average rubric score of 3 of greater
    • Percentile Example: Students will score at or above the 50th percentile
  6. Fill out the 'cutoff-score' field with the cutoff-score
    • The score students are expected to meet in order for the measure to pass
  7. Select the tool type
    • The type of tool that will be used to prove the measure
    • Rubrics can be weighted or unweighted
    • Test scores can be input individually or in batched by uploading a .csv in the correct format
  8. Select 'Add' to complete creation

  1. Find the cycle the measure is associated with (it can be a previous cycle or running cycle)
  2. Select 'Visit' by the desired cycle
  3. Select the outcome the measure is associated with
  4. Select the desired measure to view its status, edit it, or delete it

  1. Select 'Outcomes' in the navigation bar
  2. Select the desired outcome
  3. The status of each measure associated with the outcome is displayed

  1. Access the measure the evaluator will be added to
    • Check out 'Access Measure' under the Measures heading
  2. Select the desired outcome
  3. Select the '+' next to the 'Evaluators' heading
  4. Fill in the evaluator's name in the name field
  5. Fill in the evaluator's email in the Enter email field
  6. Select 'Add Evaluator'
